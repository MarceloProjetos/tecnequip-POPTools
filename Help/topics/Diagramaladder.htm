Linguagem Ladder (LD)

Linguagem Ladder:  Mesmo tendo sido a primeira linguagem destinada especificamente à programação de PLCs, a Linguagem Ladder mantém-se ainda como a mais utilizada, estando presente praticamente em todos os PLCs disponíveis no mercado. Por ser uma linguagem gráfica, baseada em símbolos semelhantes aos encontrados nos esquemas elétricos (contatos e bobinas), as possíveis diferenças existentes entre os fabricantes de PLCs, quanto à representação das instruções, são facilmente assimiladas pelos usuários, que possuem noções de diagrama de painéis elétricos.



Neste caso, a representação acima em Linguagem Ladder para os Elementos ENTRADA_X2 e ENTRADA_X1 são Contatos Normalmente Abertos e Normalmente Fechado. Porém, a localização de cada um na Lógica de Controle determina que a seguência das  Instruções sejam diferentes, ou seja, o Contato Normalmente Aberto precisa estar fechado para que o estado de ENTRADA_X1 seja relevante para a bobina saída (s).

Conforme demonstrado acima essa é uma linguagem de Diagrama de Blocos de Funções, ou seja, os programas feitos nesta linguagem, os valores fluem das entradas para as saídas, através de blocos de funções.


Composição do CLP: O hardware desenvolvido compreende um circuito lógico que pode ser dividido em:

  • Circuito lógico "CPU" - CortexM3;
  • Circuito de comunicação RS-485;
  • Circuito de comunicação Ethernet 10/100;
  • Circuitos de alimenção 8-30V CC;
  • Circuitos de entrada analogica de 0-10V;
  • Circuito de saida analogica -10V a +10V;
  • Circuitos de entrada digital de 8-30V CC;
  • Circuito de saida digital 16 portas de estado solido;
  • Circuito de gravação via USB "Obs- O Clp pode ser gravado via USB sem a necessidade de alimentação externa;
